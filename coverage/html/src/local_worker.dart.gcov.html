11/// Local workers are similar to other Workers except that they run in the context of the current thread.
12/// They do not create any platform thread (such as Isolate or Web Worker) but they provide a [channel] that
13/// can be shared with other workers to support communication between threads. One interesting use-case is
14/// accessing UI components or platform plugins in Flutter, where only code running in the main thread is
15/// allowed access to such features. Creating a [LocalWorker] in the main application and sharing its
16/// [channel] with other workers enables providing access to Flutter features.
